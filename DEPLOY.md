# 🚀 Публикация GROT — чтобы приложение работало у всех

Этот гайд проведёт от «работает у меня на ноутбуке» до «любой человек открывает по ссылке
и ставит на телефон». Дальше — путь в App Store / Google Play.

---

## Этап 1. Локальный продакшен-прогон (проверка перед публикацией)

```bash
cd grot-app
npm run prod        # соберёт фронт и запустит сервер на http://localhost:4000
```
Откройте http://localhost:4000 — это **ровно то**, что увидят пользователи. UI и API на одном домене.

---

## Этап 2. Выложить в облако (доступно всем из интернета)

Рекомендуемый путь — **Render** (просто и есть бесплатный тариф для теста).

1. Создайте репозиторий на GitHub и залейте папку `grot-app`:
   ```bash
   cd grot-app
   git init && git add . && git commit -m "GROT app"
   git branch -M main
   git remote add origin https://github.com/ВАШ_АККАУНТ/grot-app.git
   git push -u origin main
   ```
2. Зайдите на **render.com** → New → **Blueprint** → выберите репозиторий.
   Render прочитает `render.yaml` и развернёт сервис (Docker).
3. Через пару минут получите ссылку вида `https://grot-app.onrender.com`.
   **Это и есть рабочее приложение для всех.** HTTPS уже включён (нужен для PWA, SMS, Push).

> Альтернативы: **Railway** (railway.app — деплой из того же Dockerfile),
> любой **VPS** (Hetzner/DigitalOcean) через `docker build` + `docker run`.

⚠️ На бесплатном тарифе Render нет постоянного диска — данные сбросятся при «засыпании».
Для боевого бара включите платный план (`starter`) с диском (уже прописан в `render.yaml`).

## Этап 3. «Скачивание» на телефон (PWA, без магазинов)

Приложение уже устанавливается с вашей ссылки:

- **Android (Chrome):** открыть ссылку → меню ⋮ → «Установить приложение». Иконка появится на экране.
- **iPhone (Safari):** открыть ссылку → «Поделиться» → «На экран Домой».

После установки работает как обычное приложение (свой значок, полный экран, пуши).
Раздайте гостям ссылку или QR-код на неё — это и есть «скачать».

## Этап 4. Включить реальные интеграции (по ключу)

Всё работает в демо-режиме и без ключей. Чтобы включить «по-настоящему», задайте переменные
окружения в дашборде хостинга (Render → Environment). Перезапуск подхватит их автоматически.

### 🔔 Push-уведомления (бесплатно, Web Push)
```bash
npx web-push generate-vapid-keys     # выдаст Public и Private ключи
```
Задайте: `VAPID_PUBLIC`, `VAPID_PRIVATE`, `VAPID_SUBJECT=mailto:вашa@почта`.
Готово — клиенты будут получать пуши о статусе заказа на телефон.

### 📲 SMS / OTP-вход (Twilio)
1. Регистрация на twilio.com, купите номер.
2. Задайте: `TWILIO_SID`, `TWILIO_TOKEN`, `TWILIO_FROM` (ваш номер).
Вход по коду из SMS заработает автоматически (оплата — по тарифам Twilio за SMS).

### 🗺 Карты и зона доставки (Google Maps)
1. console.cloud.google.com → включите **Geocoding API**, создайте API-ключ.
2. Задайте: `GOOGLE_MAPS_KEY`, координаты бара `RESTAURANT_LAT`/`RESTAURANT_LNG`,
   радиус `DELIVERY_RADIUS_KM`.
Адреса начнут проверяться по-настоящему, расстояние и стоимость — точные.

Полный список переменных — в `.env.example`.

## Этап 5. Нативные приложения в App Store и Google Play (позже)

Когда PWA проверена на реальных гостях, оборачиваем её в нативное приложение
**без переписывания** — тем же кодом, через обёртку (Capacitor):

```bash
npm i -D @capacitor/cli @capacitor/core
npx cap init GROT com.grot.app --web-dir=frontend/dist
npx cap add ios && npx cap add android
npx cap copy
```
Дальше:
- **Android:** открыть проект в Android Studio → собрать `.aab` → загрузить в Google Play
  (аккаунт разработчика $25 единоразово).
- **iOS:** открыть в Xcode → архив → загрузить в App Store Connect
  (Apple Developer $99/год). Нужен Mac.

Модерация: Google ~1-3 дня, Apple ~1-7 дней.
