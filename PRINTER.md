# 🖨 Печать чеков (официант) — GROT

Печать чеков на термопринтер **58мм (ESC/POS)** из аккаунта официанта.
Принтер из поставки: имя **Printer001**, PIN **0000**, протокол ESC/POS.

## Что уже сделано в коде
- `frontend/src/printer.js` — ядро: сборка чека (ESC/POS) + Bluetooth-транспорт + превью-фолбэк.
- `frontend/src/screens/Staff.jsx` (`WaiterPanel`) — чип статуса «🖨 Принтер», экран выбора принтера, кнопки **«🖨 Чек»** и **«💵 Наличные»** (печать + открытие ящика) на каждом заказе, превью чека.
- Реквизиты чека (название, адрес, телефон, QR, сервисный сбор) — в `RECEIPT_CONFIG` вверху `printer.js`.

## Как это работает
- **В вебе (dev)** плагина Bluetooth нет → кнопка «Чек» показывает **превью формата** (проверять верстку чека можно без железа).
- **В Android-сборке** печать идёт на реальный принтер по Bluetooth Classic (SPP).
- ⚠️ **iOS не поддерживается** для этого принтера (дешёвые POS58 без MFi). Касса — только Android.

## Нативная часть — УЖЕ настроена ✅
Сделано в репозитории и коммитится:
- установлен плагин **`cordova-plugin-bluetooth-serial`** (Bluetooth Classic SPP → `window.bluetoothSerial`)
- создан Android-проект (`android/`), плагин подхвачен Capacitor
- в `AndroidManifest.xml` прописаны разрешения Bluetooth (legacy + Android 12+)
- веб собран с боевым API (`VITE_API_BASE=grot-app.onrender.com`) и синхронизирован

## Собрать APK (нужен Android Studio / JDK + Android SDK)
На машине с Android Studio:
```bash
cd grot-app
npm install                 # подтянет cordova-plugin-bluetooth-serial
npm run sync:app            # пересборка фронта (боевой API) + cap sync
npx cap open android        # откроется Android Studio → Build APK → на планшет
```
> ⚠️ **Android 12+**: старый плагин может не запросить разрешение `BLUETOOTH_CONNECT`
> в рантайме. Если на планшете Android 12+ подключение не идёт — выдай приложению
> разрешение «Устройства поблизости» вручную в настройках приложения, либо добавь
> рантайм-запрос. На большинстве бюджетных POS-планшетов (Android ≤ 11) работает сразу.

## Подключение принтера на планшете
1. Android → Настройки → Bluetooth → спарить **Printer001** (PIN `0000`).
2. В приложении (официант) → чип **«🖨 Принтер»** → **«Найти принтеры»** → выбрать Printer001.
3. Готово — чип станет активным. Кнопка «🖨 Чек» печатает, «💵 Наличные» — печатает и открывает ящик.

## Настройка формата чека
Правится в `RECEIPT_CONFIG` (`frontend/src/printer.js`):
- `name`, `address`, `phone` — шапка
- `qrUrl` — ссылка в QR внизу (меню / отзыв / лояльность)
- `serviceChargePct` — сервисный сбор (0 = выкл, 10 = +10%)

## Известное ограничение — язык
Принтер печатает **латиницу** из коробки. Названия блюд берутся из `nameEn`,
поэтому чеки печатаются на английском корректно. Для **тайского/кириллицы** нужно
переключение кодовой страницы принтера (`ESC t n` + соответствующая кодировка) —
см. заметку внизу `printer.js`.
